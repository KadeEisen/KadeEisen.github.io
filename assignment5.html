Challenge 5: Accessibility Concerns

By using a table, screen readers will get stuck on explaining the formatting of the table to the user instead of the acutal information present. While it is not impossible for the user to understand what is in the table, they have to sit through the screen reader informing them of the size of the table, the rows and columns, and other data useless to the user. Instead of a table, I would use CSS to create margins that organize the table in the same way. That way, the user does not need to listen through minutes of a screen reader telling them how awesomely technical my table is.
